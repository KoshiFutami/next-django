# review

## 目的

マージ前に **正しさ・セキュリティ・保守性・テスト** を確認する。コードレビュースキルがある場合はそれに沿う。

## 手順（エージェント向け）

1. 対象: 現在ブランチの `git diff main...HEAD` またはオープンな PR の差分（`gh pr diff`）。
2. **正しさ**: 要求を満たすか、境界条件・エラーハンドリングは十分か。
3. **セキュリティ**: 認証・認可・入力検証・秘密情報の混入・CSRF/CORS の意図。
4. **保守性**: 命名、レイヤー違反、重複、不要な複雑さ。
5. **テスト**: 変更に対しテストが付いているか。バックエンドなら `docker compose exec backend pytest -q` を提案。

## CLI（参考）

```bash
gh pr view
gh pr diff
make test
```

## Makefile ショートカット

```bash
make review
```

現在ブランチに紐づく PR を `gh pr view` で表示する（`gh` 必須）。
