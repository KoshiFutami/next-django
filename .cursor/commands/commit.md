# commit

## 目的

未コミットの差分を確認し、**意図ごとにコミットを分割**して履歴を読みやすくする。

## 手順（エージェント向け）

1. `git status -sb` と `git diff`（必要なら `git diff --cached`）で変更範囲を把握する。
2. 1コミットにまとめすぎない。chore / feat / fix / docs / test など、**種類が混ざるなら分割**する。
3. ステージ: `git add -p` またはファイル単位で意図どおりに追加する。
4. メッセージは **英語の type(scope): subject** またはチーム規約に従う。本文は必要なら箇条書き。
5. 実行: `git commit`（エディタ）または `git commit -m "..."`。

## Makefile ショートカット

```bash
make commit
```

ステージ済み変更があるときだけ `git commit` を起動する。未ステージなら案内を表示して終了。

## 注意

- `.env`、個人メールのデフォルト、シークレットをコミットしない。
- ユーザーが明示したコミットメッセージがあればそれを優先する。
